Green Hive 2.0 Demo

This is the login page for the Green Hive application. We allow for users of different levels of authority. Admins are given access to the full range of the site and are able to import and modify data. Basic users are only given permissions to view subsets of voter information as set by the admins.
This is the Green Hive home page which provides a simple starting point for the user. It explains the basic function of each page and gives statistics on the current data in the system.
The user can import voter data into the site from a variety of sources.
Here the user has a multitude of options in how they define their search parameters. From personal identifiers to more broad demographic questions, to focusing in on particular geographic areas. Moreover, they can define the format in which the data is displayed and which type of person is displayed in the results(voters/contacts/members/donors).
Here are the results when searching for males between the ages of 20 and 26, in area code 98225, who voted in 4 of the last 4 elections. It is currently displaying the simple format and showing all types of people who match the parameters.
Here we choose to focus in on one of the voters in the results by clicking on their state voter id. From this page we are able to view all of that voters information as well as view them on the map or edit their information.
Here is the edit voter page for that voter.
Here is the voter viewed on the map.
This is the page to add a new voter. I will add Green Hive developer Ian Barbour to the voter database as an example.
The optional information button can be pressed to show all of the additional information that can be stored for a voter, but is not required.
As an example we will add Ian Barbour as a Green Party contact.
As an example we will add Ian Barbour as a Green Party member.
As an example we will add Ian Barbour as a Green Party donor.
If the user now searches for the state voter id used for the added person then they show up in the results as a voter, member, contact and donor.
If we click on the state voter id of any of the results than all of the information on this person is congregated into a single page.
And the user is able to edit any of pages associated with that person.
This is the results of a new search for men between the ages of 20 and 24, in the 98225 area code and living on 20th street.
Then the user can create a list with a name of their choosing and add all of the results to that list.
Then they can view their defined lists on the lists page.
The user can remove users from the list as needed.
The user can print the list in a user friendly format.
Moreover, the user can view the list on the map.
We can get directions between all of the voters in the list. In this case since they are all on 20th street it is a fairly simple route. Additionally, the written directions appear under the map.
The users are also able to look at the map through a precinct view. Then the user can choose a precinct to populate with voters.
Here all the voters in precinct 249 are displayed.
If the user does not want to work with that many voters at one time they can hide all voters and draw a shape of their choosing.
They can the populate this area with all the voters in the area and work with just this subset of voters.
